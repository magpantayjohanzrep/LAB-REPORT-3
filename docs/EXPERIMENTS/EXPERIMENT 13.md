# EXPERIMENT 13 - PCM DECODING 

# OVERVIEW üîç
- Pulse Code Modulation (PCM) is a fundamental system used to convert message signals into a continuous serial stream of binary numbers, a process known as encoding. The recovery of the original message from this digital stream is referred to as decoding. At its most basic level, the decoding process requires identifying new frames within the data stream, extracting the binary numbers, and generating a proportional voltage for each value. This voltage is held until the next frame is decoded, creating a Pulse Amplitude Modulation (PAM) version of the original signal, which is then passed through a low-pass filter for final reconstruction. Successful decoding relies heavily on maintaining a clock frequency identical to the encoder's and correctly detecting the beginning of each frame to avoid data misinterpretation.

# OBJECTIVES üìù
- The primary objective of this experiment is to gain practical experience with PCM decoding using the Emona Telecoms-Trainer 101. Students will aim to convert both a sinewave and speech signals into a PCM data stream and subsequently decode them back into PAM signals. A key focus of the exercise is to understand the necessity of "stealing" the clock and frame synchronization signals from the encoder to ensure the decoder operates correctly. Additionally, the experiment objective includes recovering the original message signals using a Tuneable Low-pass filter module and comparing the auditory and visual differences between the decoded output and the original source.

# PART A TO D QUESTIONS üìù

Question 1: What does the PCM Decoder's "stepped" output tell you about the type of signal that it is?

- Answer: The "stepped" output indicates that the signal is a Pulse Amplitude Modulation (PAM) signal. The PCM decoder converts each binary number from the data stream into a proportional voltage and holds that voltage constant until the next frame is decoded. This "holding" action creates the characteristic steps, forming a PAM version of the original message before it is smoothed out by a filter.

Question 2: What must be done to the PCM Decoder module's output to reconstruct the message properly?

- Answer: To reconstruct the message properly, the decoder's output (the PAM signal) must be passed through a low-pass filter. The stepped PAM signal contains the original message plus high-frequency components (the "steps"). The low-pass filter removes these high-frequency components, smoothing the waveform to recover the original analog message.

Question 3: Even though the two signals look and sound the same, why isn't the reconstructed message a perfect copy of the original message?

- Answer: The reconstructed message is not a perfect copy of the original due to Quantization Error (or Quantization Distortion).  The manual explains that the PCM Encoder module works with 8-bit binary numbers. This means the continuous analog message (which has an infinite number of possible voltage levels) is mapped to a finite set of discrete levels.
  
- Because the encoder must round the actual voltage of the message to the nearest available binary value, there is a small discrepancy between the original signal and the digitized version. This discrepancy is the quantization error, which results in a reconstructed signal that is very close to, but not identical to, the original.

# Reflection/Learning Summary üí°

# The Digital-to-Analog Process:

- We learned that decoding is essentially a multi-step translation. It begins with identifying binary frames (synchronization), converting those bits into discrete voltage levels (quantization), and holding those levels to form a "stepped" Pulse Amplitude Modulation (PAM) signal. This helped us visualize that a "digital" signal, before final filtering, is actually a series of discrete analog steps rather than a smooth continuous curve.

# The Importance of Synchronization:

- A major takeaway was the absolute necessity of synchronization. The experiment highlighted that for the decoder to work, it must "speak the same language" as the encoder‚Äîmeaning it requires the exact same Clock frequency and Frame Synchronization (FS) signals. Without these "stolen" signals from the encoder, the decoder would misinterpret the binary stream, resulting in garbled noise rather than a coherent message.

# The Role of Filtering:

- We observed that the raw output of a PCM decoder is "blocky" or "stepped" (PAM). The experiment clarified the role of the Low-Pass Filter as a reconstruction tool. By filtering out the high-frequency components (the sharp edges of the steps), the filter smoothed the signal, restoring the original analog shape (sine wave or speech) and making it intelligible.

# Quantization Reality:

Finally, the experiment demonstrated the concept of Quantization Error. Even with correct synchronization and filtering, we realized that the reconstructed signal is never a perfect mathematical copy of the original. Because we are mapping infinite analog values to a finite 8-bit scale (256 levels), there is always a small margin of error. This was audible as subtle differences in the speech quality or visible as slight imperfections in the waveform, reinforcing the trade-offs inherent in digital communication systems.
